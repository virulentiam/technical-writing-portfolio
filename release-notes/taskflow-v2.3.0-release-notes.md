## Обзор релиза

Релиз TaskFlow v2.3.0 добавляет долгожданную функцию повторяющихся задач, улучшает производительность загрузки больших проектов и исправляет критические баги, связанные с уведомлениями. Это обновление также включает улучшенную двустороннюю интеграцию с Google Calendar и новую темную тему интерфейса для комфортной работы в условиях низкой освещенности.

**Основные изменения:**
* Повторяющиеся задачи
* Темная тема интерфейса
* Улучшенная производительность
* Исправлено 12 багов

## Новые функции

### Повторяющиеся задачи

Теперь вы можете создавать задачи, которые автоматически повторяются по заданному расписанию, что идеально подходит для рутинных операций (ежедневные отчеты, еженедельные встречи, ежемесячная оплата счетов).

**Как использовать:**
1. При создании задачи включите опцию «Повторяющаяся задача».
2. Выберите частоту повторения: ежедневно, еженедельно, ежемесячно или настройте свое расписание.
3. Задача будет автоматически создаваться в указанное время

**Поддерживаемые паттерны:**
* Каждый день / каждые N дней.
* Определенные дни недели (например, каждый понедельник и среду).
* Определенное число месяца (например, 1-го числа каждого месяца).
* Рабочие дни (пн-пт).

---

### Темная тема интерфейса

Добавлена полноценная темная тема для снижения нагрузки на глаза при работе в условиях низкой освещенности или по личным предпочтениям.

**Как включить:**
* Перейдите в **Настройки → Внешний вид → Тема**.
* Выберите «Темная», «Светлая» или «Автоматически» (следует системным настройкам вашей операционной системы).

---

### Улучшенная интеграция с Google Calendar

Двусторонняя синхронизация задач с Google Calendar стала надежнее и получила расширенную функциональность:
* Автоматическое обновление изменений в обе стороны.
* Синхронизацию статусов задач (завершенная задача в TaskFlow помечается как выполненная в Calendar).
* Поддержку повторяющихся событий.

**Настройка:**  
Зайдите в **Настройки → Интеграции → Google Calendar → Включить двустороннюю синхронизацию**.

---

### Быстрые фильтры на панели задач

Для повышения скорости работы добавлены предустановленные фильтры, доступные в один клик:
* Мои задачи на сегодня.
* Просроченные задачи.
* Задачи без исполнителя.
* Высокоприоритетные задачи.

Фильтры доступны в левой боковой панели в разделе «Быстрые фильтры».

## Улучшения

### Производительность

* **Ускорена загрузка больших проектов.** Проекты с 500+ задачами теперь загружаются на 60% быстрее.
* **Оптимизирована работа с вложениями.** Загрузка файлов до 50 МБ происходит в 2 раза быстрее.
* **Улучшена производительность поиска.** Поиск по задачам работает на 40% быстрее.

### Интерфейс

* Переработан дизайн формы создания задачи — теперь более компактный и удобный.
* Добавлены горячие клавиши для быстрого создания задачи (`Ctrl+N` / `Cmd+N`).
* Улучшена читаемость комментариев в задачах (увеличен контраст и размер шрифта).
* Добавлена возможность сворачивать/разворачивать описания задач в списке.

### Уведомления

* Улучшена группировка уведомлений — теперь похожие уведомления объединяются.
* Добавлена возможность отложить уведомление на определенное время.
* Улучшены email-уведомления — более читаемый формат и лучшая структура.

### Мобильное приложение

* Добавлена поддержка офлайн-режима — теперь можно просматривать и редактировать задачи без интернета.
* Улучшена синхронизация данных между устройствами.
* Оптимизирован расход батареи (снижено фоновое потребление на 15%).

## Известные проблемы

* **[#2420]** В Safari 15.x может наблюдаться задержка при перетаскивании задач между колонками. **Workaround:** Используйте изменение статуса через выпадающее меню.
* **[#2433]** Экспорт проектов с более чем 1000 задач может занимать до 2 минут. Оптимизация запланирована на версию 2.4.0.
* **[#2441]** Уведомления в мобильном приложении могут приходить с задержкой до 5 минут при нестабильном интернет-соединении.

## Изменения для разработчиков

### API Changes

Этот раздел описывает изменения в **REST API TaskFlow v1** (Базовый URL: `https://api.taskflow.example.com/v1`).

#### Новые endpoints

* `POST /api/v1/tasks/recurring` — создание повторяющейся задачи.
* `GET /api/v1/tasks/recurring/{id}` — получение информации о повторяющейся задаче.
* `DELETE /api/v1/tasks/recurring/{id}` — удаление повторяющейся задачи.

#### Изменения в существующих endpoints

* `POST /api/v1/tasks` теперь поддерживает параметр `recurrence` для создания повторяющихся задач.
* `GET /api/v1/projects/{id}/tasks` добавлен параметр `include_recurring=true` для включения повторяющихся задач в ответ.

### Breaking Changes

Отсутствуют.

### Deprecations

Следующие endpoints будут удалены в версии 3.0.0 (планируемый релиз: март 2026):
* `GET /api/v1/users/me/notifications` (используйте `/api/v2/notifications` вместо этого).

## Обновление

### Для пользователей веб-версии

Обновление произойдет автоматически. Просто перезагрузите страницу (F5 или `Cmd+R`).

### Для пользователей мобильного приложения

Обновите приложение через:
* **iOS:** App Store → Обновления → TaskFlow.
* **Android:** Google Play → Мои приложения → TaskFlow → Обновить.

### Для пользователей self-hosted версии

```bash
# Остановите текущий контейнер
docker stop taskflow

# Загрузите новый образ
docker pull taskflow/taskflow:2.3.0

# Запустите обновленную версию
docker run -d --name taskflow taskflow/taskflow:2.3.0
```

**Примечание:** рекомендуем сделать резервную копию базы данных перед обновлением.
